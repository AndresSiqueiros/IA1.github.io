01 · Dataset

Descripción del Dataset

Se utilizó el archivo datos_limpios_2024.csv con registros horarios de calidad del aire y condiciones meteorológicas. El dataset contiene 8,756 filas y 14 columnas indexadas por marca de tiempo (datetime).

import pandas as pd import numpy as np import matplotlib.pyplot as plt import seaborn as sns from sklearn.model_selection import train_test_split, StratifiedKFold, cross_validate from sklearn.linear_model import LogisticRegression from sklearn.preprocessing import StandardScaler from sklearn.pipeline import Pipeline from sklearn.metrics import ( confusion_matrix, classification_report, roc_auc_score, roc_curve, accuracy_score, precision_score, recall_score, f1_score ) df = pd.read_csv('datos_limpios_2024.csv', parse_dates=['datetime']) df = df.set_index('datetime') print(f'Dimensiones: {df.shape}') # (8756, 14)

02 · Variable Objetivo

Balance de Clases

Se definió un umbral de 45 µg/m³ para clasificar cada hora como Calidad Aceptable (0) o Contaminación Alta (1).

UMBRAL = 45 df['PM25_alto'] = (df['PM25'] > UMBRAL).astype(int)

Implicación del desbalance: La exactitud (accuracy) sola es una métrica engañosa. Se priorizan Precision, Recall y F1 para la clase minoritaria, y se usa stratify=y en el split para preservar las proporciones.

División Entrenamiento / Prueba

features = ['CO', 'NO2', 'O3', 'TMP', 'HR', 'WSP', 'hora_sin', 'hora_cos', 'mes_sin', 'mes_cos', 'dia_semana'] X = df[features] y = df['PM25_alto'] X_train, X_test, y_train, y_test = train_test_split( X, y, test_size=0.20, random_state=42, stratify=y )

03 · Modelo

Pipeline: Escalado + Regresión Logística

Se construyó un Pipeline de scikit-learn que encadena el escalado de variables y el clasificador, evitando fuga de datos (data leakage) al aplicar la transformación solo con estadísticas del conjunto de entrenamiento.

pipeline = Pipeline([ ('scaler', StandardScaler()), ('clf', LogisticRegression(max_iter=1000, random_state=42)) ])
¿Por qué StandardScaler? La regresión logística optimiza con gradiente descendente, por lo que las variables deben estar en escalas comparables. El escalado también permite interpretar los coeficientes directamente en términos de importancia relativa.

06 · Análisis de Umbral

Efecto del Umbral de Decisión

El umbral por defecto (0.5) no es óptimo en problemas desbalanceados. Se evaluaron umbrales de 0.10 a 0.90 para entender el trade-off Precision–Recall.

umbrales = np.arange(0.1, 0.91, 0.05) for u in umbrales: y_pred_u = (y_prob >= u).astype(int) p = precision_score(y_test, y_pred_u, zero_division=0) r = recall_score(y_test, y_pred_u, zero_division=0) f = f1_score(y_test, y_pred_u, zero_division=0)

07 · Curva ROC

AUC-ROC = 0.8948

La Curva ROC (Receiver Operating Characteristic) muestra el trade-off entre la Tasa de Verdaderos Positivos (TPR/Recall) y la Tasa de Falsos Positivos (FPR) para todos los umbrales posibles.

Interpretación del AUC: Un AUC de 0.8948 significa que, ante un par aleatorio de un caso positivo y uno negativo, el modelo asigna mayor probabilidad al positivo el 89.5% de las veces. Es una métrica robusta porque es independiente del umbral de decisión.

09 · Conclusiones

Conclusiones

  • 🎯
    El modelo discrimina bien pero predice poco la clase minoritaria. Con AUC-ROC de 0.89 el modelo tiene excelente capacidad de separación, pero con umbral 0.5 solo detecta el 23% de los episodios peligrosos. Accuracy alta (95%) es engañosa en datasets desbalanceados.
  • ⚖️
    Validación cruzada confirma generalización sólida. La diferencia entre CV (AUC=0.9011) y prueba (AUC=0.8948) es mínima (Δ=0.006), indicando que el modelo no tiene sobreajuste y generaliza bien a datos nuevos.
  • 🔧
    El umbral de decisión es un parámetro crítico. Reducir el umbral a ~0.25–0.30 duplica el Recall (de 0.23 a ~0.51) a costa de más falsas alarmas. Para alertas de salud pública, un Recall mayor es preferible al costo de Precision.
  • 🌡️
    Temperatura y NO2 dominan la predicción. TMP es el predictor más fuerte (OR≈4.9), lo cual tiene sentido físico: las inversiones térmicas en días calurosos atrapan contaminantes. NO2 sugiere influencia del tráfico vehicular.
  • 💨
    El viento actúa como factor protector. WSP tiene coeficiente negativo, confirmando que la velocidad del viento favorece la dispersión de PM2.5 y reduce el riesgo de episodios críticos.
  • 🚀
    Próximos pasos recomendados: Explorar técnicas de balanceo (SMOTE, class_weight='balanced'), modelos no lineales (Random Forest, XGBoost), y ajuste fino del umbral con la curva Precision-Recall para maximizar el F1 en la clase 1.
